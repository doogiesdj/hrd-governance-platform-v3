운영

  • 현재 기능 목록 (구현됨 / 미구현 / 부분 구현) 정리

0-2

개발 환경 구성 및 로컬 실행 환경 표준화

인프라

현재 플랫폼은 정적 파일이라 로컬에서 file:// 프로토콜로도 동작하지만, Phase 2 이후 백엔드 도입 시 환경이 필요합니다.

체크리스트
  • Node.js 개발 서버(예: http-server, live-server) 또는 Python SimpleHTTPServer 표준화
  • .env.example 파일 생성 (Claude API key, Gemini API key 환경변수 정의)
  • API 키를 localStorage에서 환경 변수 기반으로 로드하도록 수정 준비
  • Git 브랜치 전략 수립 (main / develop / feature/* 구조)
  • GitHub Actions CI 기본 워크플로우 설정 (정적 분석, 빌드 검증)
산출물.env.example, .github/workflows/ci.yml, README.md 갱신

온톨로지 품질 검증 및 정제

데이터

체크리스트
  • Protégé(Desktop)에서 HRD_Governance_Extended_v2.rdf 열기 및 OWL DL 일관성 검사(HermiT/Pellet)

구현 절차

  • 역량 뷰(view-competency)에 히트맵 컨테이너 추가
  • 셀 hover 시 툴팁: 기관명, 역량명, 갭 수, 심각도 표시

// 기관 × 역량 갭 집계 (app.js 또는 별도 heatmap.js) function buildGapMatrix(gaps, orgs, competencies) { const matrix = {}; gaps.forEach(g => { const key = `${g.organizationId}_${g.competencyId}`; matrix[key] = (matrix[key] || 0) + 1; }); return matrix; }

프로그램-역량 정합도 점수 계산 및 표시

데이터 · 프론트엔드

190개 프로그램이 현재 역량 갭 해소에 얼마나 기여하는지 정량적으로 보여줍니다. 프로그램 선택 우선순위 결정에 직접 활용됩니다.

구현 절차

  • 프로그램 뷰에 정합도 점수 컬럼 및 정렬 기능 추가
  • 상위 10개 프로그램 "갭 해소 추천" 카드 섹션 생성

개인화된 역량 개발 로드맵 생성기

AI · 프론트엔드

개인(HumanResource)을 선택하면 AI + 온톨로지 추론이 결합하여 맞춤 학습 경로를 자동 생성합니다.

  • 개인 프로파일 조회: 현재 역량, 역량 갭, 목표 직위/직무 (SPARQL)
  • 갭 심각도 및 시장 수요 기반 우선순위 정렬 알고리즘 작성
  • 각 갭에 대해 developsCompetency 역방향 조회로 적합 프로그램 목록 생성
  • 프로그램 간 선수 학습 관계(prerequisite) 온톨로지 속성 추가 후 위상 정렬(topological sort)로 학습 순서 결정
  • 학습 로드맵을 타임라인 시각화 (D3.js Gantt 스타일)
  • 로드맵을 PDF/HTML로 내보내기 기능
노력: 높음
3-2

정책 임팩트 시뮬레이션 (What-If Analysis)

AI · 분석

정책 입안자가 파라미터를 조정하면 예상 결과가 실시간으로 시뮬레이션됩니다.

  • 시뮬레이션 모델 설계: 예산 증가율 → 수용 인원 증가 → 역량 갭 해소 수 추정 수식 정의
  • Python 시뮬레이션 엔진 구현 (FastAPI 엔드포인트로 노출)
  • 프론트엔드 슬라이더 UI: 예산 증감률, 프로그램 확대 수, 목표 연도
  • 시뮬레이션 결과: 갭 해소율, ROI, 예상 취업 증가 인원을 Chart.js로 표시
  • 시나리오 저장 및 비교 기능 (최대 3개 시나리오 병렬 비교)
노력: 높음
3-3

설명 가능한 AI 추천 (Explainable AI)

AI · 프론트엔드

AI 추천의 근거를 온톨로지 경로로 시각적으로 설명합니다. "왜 이 프로그램을 추천했는가?"에 대한 투명한 답변을 제공합니다.

  • 추천 생성 시 근거 경로 추출: (Person → Gap → Competency → Program) SPARQL 경로 쿼리
  • D3.js로 추천 근거 미니 그래프 렌더링 (상세 패널에 포함)
  • 각 엣지에 관계 레이블 표시 (한국어)
  • 관련 정책 및 예산 연결 노드까지 확장 표시
  • 이 경로에서 "약한 고리" (예: 예산 부족 구간) 하이라이트
노력: 중간

역할 기반 뷰 및 개인화 대시보드

프론트엔드

사용자 역할에 따라 다른 초기 화면과 맞춤 정보를 제공합니다.

역할기본 대시보드특화 기능

프로그램 담당자프로그램 정합도 순위수강 등록 관리, 이수율 추적
연구자/분석가전체 그래프 뷰SPARQL 직접 쿼리, 데이터 내보내기

4

Phase 4 · 생태계 연계 (Ecosystem Integration)

외부 데이터 연동 · 표준화 · 기간: 6 ~ 12개월
장기 목표
4-1

NCS(국가직무능력표준) 온톨로지 매핑

데이터 · 온톨로지

HRD 온톨로지의 Competency 클래스를 NCS 표준 직무역량과 owl:sameAs로 연결하여 국가 표준과의 정합성을 확보합니다.

  • NCS 포털(ncs.go.kr) API 또는 공개 데이터로 NCS 역량 체계 수집
  • 문자열 유사도(cosine similarity) 기반 자동 매핑 후 수동 검수
  • 매핑 결과를 owl:sameAs 트리플로 RDF에 추가
  • NCS 코드로 역량 검색 기능 추가 (코드 참조 뷰 연동)
노력: 높음
4-2

워크넷 채용공고 데이터 연계 (실시간 시장 수요 반영)

데이터 · 백엔드

채용공고에서 요구하는 직무 역량을 주기적으로 수집하여 "시장 수요 역량"으로 온톨로지에 반영합니다.

  • 워크넷 오픈 API 연동 모듈 개발 (Python requests + 공공데이터포털 API 키)
  • 채용공고 직무 키워드 → 온톨로지 역량 클래스 매핑 AI 분류기 구현
  • 주 1회 자동 수집 스케줄러 설정 (cron + Python)
  • "시장 수요 역량" vs "현재 역량 갭" 비교 뷰 추가
  • 채용 수요 증가 역량에 "급증" 태그 표시
대안워크넷 API 접근이 어려울 경우 공공데이터포털의 "국가 직업 정보" 데이터셋 활용
노력: 높음
4-3

모바일 반응형 레이아웃 및 접근성(WCAG) 개선

디자인 · 프론트엔드

정책 담당자의 현장 태블릿 사용을 고려한 반응형 레이아웃과 WCAG 2.1 AA 준수로 접근성을 개선합니다.

  • CSS media query 추가: 768px 이하 사이드바 하단 탭 네비게이션으로 전환
  • 색상 대비 비율 4.5:1 이상으로 다크 테마 팔레트 조정
  • 키보드 탐색 지원: tabindex, aria-label, role 속성 추가
  • 그래프 시각화의 대체 텍스트 및 데이터 테이블 형식 토글 제공
  • Lighthouse 접근성 점수 90+ 달성 목표
노력: 중간
4-4

온톨로지 구동 UI (Ontology-Driven UI) 구현

프론트엔드 · 온톨로지

온톨로지에 UI 메타데이터 어노테이션을 추가하여 새 클래스/속성 추가 시 UI가 자동으로 업데이트되게 합니다.

  • 커스텀 OWL 어노테이션 속성 정의: ui:defaultView, ui:groupBy, ui:chartType
  • 온톨로지에서 UI 메타데이터 SPARQL로 조회하는 함수 작성
  • 클래스별 기본 뷰 유형 자동 결정 로직 구현 (숫자 속성 → 차트, 관계 → 그래프 등)
  • 새 RDF 클래스 추가 시 자동 사이드바 탭 생성 메커니즘 구현
난이도 주의이 기능은 가장 복잡한 구현 중 하나입니다. Phase 4의 다른 작업이 완료된 후 마지막에 구현을 권장합니다.
노력: 매우 높음
